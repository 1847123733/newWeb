# Node.js 三层架构 Web 服务

基于 Express 的 Node.js 后端服务，采用类似 Java 的三层架构设计模式。

## 技术栈

| 依赖 | 版本 | 说明 |
|------|------|------|
| express | ^4.18.2 | Web 框架 |
| mysql2 | ^3.6.5 | MySQL 数据库驱动 |
| jsonwebtoken | ^9.0.2 | JWT 认证 |
| cookie-parser | ^1.4.6 | Cookie 解析 |
| morgan | ^1.10.0 | HTTP 请求日志 |
| dotenv | ^16.3.1 | 环境变量管理 |

## 项目结构

```
newWebServe/
├── config/
│   └── database.js           # 数据库连接池配置
├── models/                    # 数据模型层（Entity）
│   ├── UserInfo.js           # 用户信息实体
│   ├── UserVisitInfo.js      # 用户访问信息实体
│   └── UserConfig.js         # 用户配置实体
├── dao/                       # 数据访问层（DAO）
│   ├── userInfoDao.js        # 用户信息数据访问
│   ├── userVisitDao.js       # 用户访问数据访问
│   └── userConfigDao.js      # 用户配置数据访问
├── services/                  # 业务逻辑层（Service）
│   ├── authService.js        # 认证服务
│   ├── visitService.js       # 访问记录服务
│   └── configService.js      # 用户配置服务
├── controllers/               # 控制器层（Controller）
│   ├── authController.js     # 认证控制器
│   ├── visitController.js    # 访问记录控制器
│   └── configController.js   # 用户配置控制器
├── routes/                    # 路由层
│   ├── auth.js               # 认证路由
│   ├── visit.js              # 访问记录路由
│   └── config.js             # 用户配置路由
├── middleware/                # 中间件
│   ├── auth.js               # JWT 认证中间件
│   └── validator.js          # 参数验证中间件
├── utils/                     # 工具类
│   ├── response.js           # 统一响应封装
│   └── jwt.js                # JWT 工具类
├── my.sql                     # 数据库表结构
├── .env                       # 环境变量配置
├── server.js                  # 应用入口
└── package.json               # 项目配置
```

## 快速开始

### 1. 安装依赖

```bash
npm install
```

### 2. 配置环境变量

编辑 `.env` 文件：

```env
PORT=3000
NODE_ENV=development

DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=your_database

JWT_SECRET=your_jwt_secret_key_here
JWT_EXPIRES_IN=7d
```

### 3. 创建数据库表

执行 `my.sql` 中的 SQL 语句创建表结构。

### 4. 启动服务

```bash
npm run dev   # 开发模式
npm start     # 生产模式
```

## API 接口

### 认证接口 `/api/auth`

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | /register | 用户注册 | ❌ |
| POST | /login | 用户登录 | ❌ |
| POST | /logout | 用户登出 | ❌ |
| GET | /me | 获取当前用户信息 | ✅ |

### 用户访问接口 `/api/visit`

| 方法 | 路径 | 说明 | 认证 | 权限 |
|------|------|------|------|------|
| GET | / | 查询访问记录列表 | ✅ | 超级管理员 |
| POST | / | 新增访问记录 | ✅ | - |

### 用户配置接口 `/api/config`

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| GET | / | 查询用户配置 | ✅ |
| POST | / | 新增用户配置 | ✅ |
| PUT | / | 修改用户配置 | ✅ |

## 请求示例

### 注册

```bash
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"userName": "测试用户", "userAccount": "testuser", "userPassword": "123456"}'
```

### 登录

```bash
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"userAccount": "testuser", "userPassword": "123456"}'
```

### 获取配置

```bash
curl -X GET http://localhost:3000/api/config \
  -H "Authorization: Bearer <token>"
```

## 响应格式

### 成功响应

```json
{
  "code": 0,
  "message": "操作成功",
  "data": { ... }
}
```

### 错误响应

```json
{
  "code": -1,
  "message": "错误信息",
  "data": null
}
```

## 架构流程图

```
┌─────────────────────────────────────────────────┐
│              前端（Vue/React）                   │
└─────────────────┬───────────────────────────────┘
                  │ HTTP 请求
                  ↓
┌─────────────────────────────────────────────────┐
│  路由层 (Routes)                                 │
│  - routes/auth.js                               │
│  职责: 定义 API 端点，映射到 Controller             │
└─────────────────┬───────────────────────────────┘
                  ↓
┌─────────────────────────────────────────────────┐
│  控制器层 (Controllers)                          │
│  - authController.js                            │
│  职责: 处理 HTTP 请求/响应，参数验证，调用 Service│
└─────────────────┬───────────────────────────────┘
                  ↓
┌─────────────────────────────────────────────────┐
│  业务逻辑层 (Services)                           │
│  - authService.js                               │
│  职责: 核心业务逻辑，事务处理，数据转换           │
└─────────────────┬───────────────────────────────┘
                  ↓
┌─────────────────────────────────────────────────┐
│  数据访问层 (DAO)                                │
│  - userInfoDao.js                               │
│  职责: 数据库操作（CRUD），SQL 语句              │
└─────────────────┬───────────────────────────────┘
                  ↓
┌─────────────────────────────────────────────────┐
│  实体层 (Models)                                 │
│  - UserInfo.js                                  │
│  职责: 数据模型定义，数据验证，对象转换           │
└─────────────────┬───────────────────────────────┘
                  ↓
┌─────────────────────────────────────────────────┐
│             MySQL 数据库                         │
└─────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────┐
│ 前端                                                          │
└──────────────────────────────┬──────────────────────────────┘
                               │ HTTP 请求
                               ↓
┌─────────────────────────────────────────────────────────────┐
│ Controller 层 (@RestController)                             │
│   • 接收请求 (@RequestMapping / @GetMapping 等)             │
│   • 参数校验 (@Valid + @RequestBody / @RequestParam)       │
│   • 全局异常处理 (@ControllerAdvice)                        │
│   • 返回统一响应 (Result / R 对象)                          │
│   示例：/login/getLoginUser                                 │
└──────────────────────────────┬──────────────────────────────┘
                               │ 调用 Service
                               ↓
┌─────────────────────────────────────────────────────────────┐
│ Service 层 (@Service)                                       │
│   • 核心业务逻辑处理                                        │
│   • 事务控制 (@Transactional)                               │
│   • 参数校验、权限检查、业务规则                            │
│   • 调用  Mapper（sql）                                │
│   • 常见：Service 接口 + ServiceImpl 实现类                 │
└──────────────────────────────┬──────────────────────────────┘
                               │ 调用 Mapper sql
                               ↓
┌─────────────────────────────────────────────────────────────┐
│ Mapper  层 (MyBatis )                                         │
│   • 数据访问层                                              │
│   • MyBatis：@Mapper + XML / 注解方式                       │
│   • CRUD 操作、复杂查询 (Pageable 分页、@Query 等)         │
└──────────────────────────────┬──────────────────────────────┘
                               │ 操作数据库
                               ↓
┌─────────────────────────────────────────────────────────────┐
│ MySQL 数据库                                                │
└─────────────────────────────────────────────────────────────┘
```
